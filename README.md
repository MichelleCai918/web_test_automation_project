# web_test_automation_project
This project represents my transition from manual QA testing to test automation.

---

## Introduation
I previously performed manual testing on web and mobile applications. Through this project, I started learning how to translate manual test cases into simple automated tests using Python.


## Reasons for this project
While working as a QA tester, I realized automation is essential for improving efficiency and maintaining product quality. This project is my first step toward building automation skills.


## Project scope & contents
Manual test suites, Issue tracking samples, and Automated scripts
 1. Manual Test Documentation

    Core User Flows: Comprehensive test cases for Authentication (Login) and Data Entry (Form Submission).

    Test Case Library: Detailed step-by-step instructions, expected results, and test data sets.

2. Defect & Issue Management

    Standardized Bug Reports: High-quality defect logs including precise reproduction steps.

    Error Analysis: Detailed documentation of observed vs. expected behaviors with visual evidence.

3. Automation Framework 

    Python Test Scripts: Initial automated sequences for critical path regression. This project also includes a simple Python script to generate test cases automatically.

Run:

python generate_test_cases.py "user login"

    Architecture Foundation: Scalable folder structure and configuration files for future test expansion.

## Technical Competencies Acquired
Through this project, I have focused on Test Logic Translation, effectively mapping complex manual scenarios into robust, and machine-executable automation scripts. I am building a strong foundation in Automation Architecture by establishing organized, Python-based testing frameworks that prioritize scalability. Furthermore, I have prioritized Technical Documentation, mastering the art of creating clear and maintainable test suites that bridge the gap between human readability, and automated precision.

*  Built a foundation in automation architecture by organizing Python-based testing workflows with scalability in mind
